## Project Overview

This is the official website for the Cal State LA University-Student Union (U-SU) — a student-facing portal for programs, events, services, and campus resources. Built with Next.js 14 (Pages Router) and deployed on Vercel.

## Commands

```bash
yarn dev          # Dev server (Next.js + next-video sync)
yarn build        # Production build (also generates sitemap)
yarn start        # Production server
yarn lint         # ESLint via next lint
yarn ts-lint      # TypeScript type checking (incremental watch)
yarn dev:sections # Dev server with section schema generation watching
```

Node version: 22.18.0 (use `nvm use`). Package manager: Yarn.

## Architecture

### Routing & Pages

Next.js Pages Router with file-based routing under `src/pages/`. API routes live at `src/pages/api/` and integrate with JotForm, Notion, Supabase, and Instagram. TypeScript `baseUrl` is `./src`, so imports are relative to that root.

### Component Organization

- `src/components/` — reusable UI primitives (Button, Card, Modal, Toast, etc.)
- `src/modules/` — feature-based compositions (EventCard, BoardOfDirectors, Backoffice, etc.)
- `src/sections/` — full-width page sections (Hero, CTA, Instagram feed, etc.) with JSON schemas generated by `yarn generate-section-schemas`
- `src/partials/` — partial page layouts

### Styling

styled-components (SSR-enabled via `next.config.js`) + styled-system for spacing/layout props. Theme tokens live in `src/theme/`.

### State Management

- **Recoil** atoms in `src/atoms/` for global state (search results, events, editor, JotForm submissions, graphics requests)
- **React Context** (`src/context/ToastContext`) for toast notifications
- **NextAuth SessionProvider** for auth state

### Authentication & Authorization

NextAuth.js with Azure AD provider. `src/middleware.ts` protects `/backoffice` and `/graffix/backoffice` routes. Supabase maintains an allowlist of permitted backoffice users. Auth middleware uses JWT validation from next-auth/jwt.

### External Integrations

| Service          | Purpose                                                    |
| ---------------- | ---------------------------------------------------------- |
| Notion API       | Graphics requests, work orders, name tags (3 separate DBs) |
| JotForm          | Grad submissions, contact forms, U-Krew sign-up            |
| Supabase         | Backoffice user allowlist                                  |
| Upstash Redis    | Rate limiting via @upstash/ratelimit                       |
| MUX / next-video | Video assets                                               |
| Instagram        | Feed on homepage                                           |

All credentials are injected via `.env.local` — see README for the full variable list.
